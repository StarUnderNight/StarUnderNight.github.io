ThreadPool

一、Excutor框架
1.1 this逃逸问题
this 逃逸是指在构造函数返回之前其他线程就持有该对象的引用. 调用尚未构造完全的对象的方法可能引发令人疑惑的错误。

1.2 Executor接口
Executor是一个顶层接口，在它里面只声明了一个方法execute(Runnable)，返回值为void，参数为Runnable类型，从字面意思可以理解，就是用来执行传进去的任务的；

1.3 ExecutorService接口
ExecutorService接口继承了Executor接口，并声明了一些方法：submit、invokeAll、invokeAny以及shutDown等
shutdownNow(); // 立即关闭所有正在执行的任务，并返回待执行的任务列表

1.4 AbstractExecutorService抽象类
抽象类AbstractExecutorService实现了ExecutorService接口，基本实现了ExecutorService中声明的所有方法

1.5 ThreadPoolExecutor
然后ThreadPoolExecutor继承了类AbstractExecutorService
多个构造参数

1.6 Executor类
工厂类，提供了特定的线程池的创建

  • ExecutorService：真正的线程池接口。
  • ScheduledExecutorService：能和Timer/TimerTask类似，解决那些需要任务重复执行的问题。
  • ThreadPoolExecutor：ExecutorService的默认实现。
  • ScheduledThreadPoolExecutor：继承ThreadPoolExecutor的ScheduledExecutorService接口实现，周期性任务调度的类实现。

一些问题：
1）run方法和start方法的区别
run相当于调用线程对象的普通方法；start是启动一个线程，然后由JVM去调度执行这个线程的代码
2）线程的调度方式
抢占式和分时；java采用的抢占式，共有10个优先级，默认5，最高10，最低1；
3）为什么jdk1.5增加了Callable
Callable可以返回结果以及抛出异常

http

HTTP发展历程

一、HTTP基本结构
浏览器

二、HTTP1.0 和1.1区别
1、缓存处理
HTTP1.0
If-Modified-Since：请求头标签，浏览器把某个缓存页面中的最后修改时间发给服务器，服务器做比较，如果没有改变，则返回304，客户端收到后，就会把本地缓存的页面显示到浏览器
Expires：如果某个缓存文件超过这个时间，则需要重新到服务器去请求这个资源；如果没有超时，则直接使用这个缓存中的文件
HTTP1.1
Entity tag
If-Unmodified-Since：与If-Modified-Since
If-Match：这个值和ETge一致时，服务器才会处理这个请求
If-None-Match：当这个值和ETge不一致时，服务器处理该请求
ETge：唯一标识某个资源，资源更新时，也会更新ETge；又分为强弱ETage；比如下载过程中出现中断，则会根据ETge来指定资源

2、带宽优化以及网络连接使用
HTTP1.1在请求头引入range字段，允许请求资源某个部分；比如 Range:bytes=5001-100000，请求5001到100000的字节资源，服务器返回206，如果无法处理请求，则返回200以及全部资源

3、错误通知的管理
HTTP1.1 新增24个错误状态码

4、Host处理
HTTP1.1 支持Host字段，同一个ip可以设置多个站点，用host区区别多个站点

5、长连接
HTTP1.1支持长连接和请求的流水线处理，一个TCP连接上可以传送多个HTTP请求和响应；本质就是复用TCP连接

三、HTTP2.0
1、协议解析采用二进制格式
2、多路复用：多个请求可以放在一个TCP连接上，服务器通过请求的id来区别，并把这些请求归属到不同的服务端请求中
3、header压缩：客户端和服务端缓存一个头部字段表，避免header重复
4、服务端推送：相当于我请求了a.html，你把a.css和a.js都推送给我了，客户端不用再请求a.css和a.js
